Export bugs to fix:

  Current state:
  - Export Full Overlay PNG works correctly.
  - Export Right Sidebar PNG downloads a file, but the PNG is blank/empty.
  - Export Bottom Bar PNG downloads a file, but the PNG is blank/empty.
  - Export Cover PNG does not work.

  Please fix the export implementation only. Do not change the visual design unless necessary.

  Root-cause expectations:
  The right sidebar and bottom bar export likely point to the wrong DOM node, a scaled/transformed preview node, or a wrapper without rendered content. Cover export likely fails because the cover canvas is
  not mounted when the Overlay tab is active, or its ref is null.

  Requirements:
  1. Full overlay export:
     - Keep existing behavior.
     - Export exact 1920x1080 PNG.

  2. Right sidebar export:
     - Export only the right sidebar content.
     - Output size should match the actual sidebar dimensions:
       width: 470px
       height: 760px
     - The image must include the sidebar panel and all text.
     - It must not be blank.
     - It must not include the main screen or bottom bar.

  3. Bottom bar export:
     - Export only the bottom status bar content.
     - Output size should match:
       width: 1856px
       height: 180px
     - The image must include the bottom bar panel and all text.
     - It must not be blank.
     - It must not include the main screen or sidebar.

  4. Cover export:
     - Export the cover canvas at 1920x1080.
     - It should work even if the user is currently on the Overlay tab.
     - If the Cover tab/canvas is not mounted, render an offscreen cover export node or temporarily mount it for export.
     - It must not fail silently.

  Implementation guidance:
  Use one of these robust approaches.

  Preferred approach:
  Create hidden, unscaled export-only DOM nodes:
  - ExportOverlayCanvas
  - ExportSidebarOnly
  - ExportBottomBarOnly
  - ExportCoverCanvas

  These nodes should:
  - be rendered offscreen, not display:none
  - use position: fixed
  - left: -10000px
  - top: 0
  - width/height equal to export size
  - contain the same components/content as the preview
  - have no CSS transform or preview scaling

  Example CSS:
  .export-stage {
    position: fixed;
    left: -10000px;
    top: 0;
    pointer-events: none;
    opacity: 1;
  }

  Do not use display: none because html-to-image cannot render hidden nodes.

  Then each export button should call html-to-image on the corresponding unscaled export node:
  - full overlay -> 1920x1080 node
  - right sidebar -> 470x760 node
  - bottom bar -> 1856x180 node
  - cover -> 1920x1080 node

  Alternative approach:
  Use html-to-image on the full 1920x1080 overlay node and crop by canvas after rendering:
  - right sidebar crop: x=1410, y=48, width=470, height=760
  - bottom bar crop: x=24, y=840, width=1856, height=180
  But export-only DOM nodes are cleaner.

  Important:
  - Do not export from the scaled preview node.
  - Do not export from a wrapper that only controls layout.
  - Do not use nodes with overflow clipping that hides the target.
  - Make sure all fonts/images are loaded before export.
  - Add try/catch and show a visible error message if export fails.

  Acceptance tests:
  - Export Full Overlay PNG opens and has complete overlay.
  - Export Right Sidebar PNG opens and contains only the sidebar content.
  - Export Bottom Bar PNG opens and contains only the bottom bar content.
  - Export Cover PNG opens and contains the cover at 1920x1080.
  - All files have non-zero dimensions and visible content.
